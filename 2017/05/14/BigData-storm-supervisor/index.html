storm supervisor 的实现

1. 保存 supervisor id

创建 {storm-local}/“isupervisor” 目录。以 “supervisor-id” 作为 key 。以 .version 为文件。这个文件中存储着 SupervisorId. 这个 SupervisorId 就是一个 uuid, 使用 UUID.randomUUID 方法生成。

long-num，也就是文件名称的生成使用 当前系统的毫秒数，此后就使用这个毫秒数+1的方法生成 version.

核心参考：org.apache.storm.utils.LocalState。

提供了一种基于版本的状态存储机制。使用文件在存储一个 Map<String, TBase> 的 map 对象。每次向 Map 中添加（当然也包括修改）元素的时候，这个 Map 的版本号就会增加1. 这个文件存储在 {storm-local}/“isupervisor” 目录中。文件的名称是 .version, 此后，增加版本号，则新创建文件 <timestamep+1>.version
